Rules

In Eugene, we categorize the rules into the following types of constraints: counting, positional, pairing. To formally define the semantics of the rules, we define N as the length/size of the genetic design.


Counting Rules

Counting rules constraint the number of occurrences of specific genetic elements in a design. We define the cardinality of genetic elements as follows: |α| is the number of occurrences of α in the design.

Positional rules constraint the location of occurrence of genetic elements. To formalize the semantics of the positional rule, we define the position predicate as position(α) = i with 1 ≤ i ≤ N. α can only appear on non-constraint positions in the genetic design.

Paring Rules

Pairing rules constrain the pair-wise appearance of genetic elements. For example, various elements cannot occur in a genetic system by themselves.
