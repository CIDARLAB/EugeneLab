Control Flow Statements

Eugene scripts are generally executed in a top-down manner. Control-flow statements break up the flow of execution and allow blocks of code to be executed conditionally or as long as a condition is fulfilled. Conditions are value comparisons of primitives, biological properties and components, as well as evaluations of rules. For value comparisons, Eugene offers the following relational operators:
• == ... equal to
• != ... not equal to
• > … greater than
• >= ... greater or equal to
• <= ... less or equal than
• < … less than

In Eugene, only values of the same type can be compared. If the values are arrays, then only the == and != operators are supported, where each element in one list is compared to the element at the corresponding position in the other list.
Conditions can be:
• combined by using the AND, OR, or XOR logical operators,
• negated by using the NOT operator, and
• grouped by using parenthesis.

IF-ELSE Statements

Eugene offers conditional branches (i.e., IF-ELSE statements) to manage the execution of specific actions under certain conditions. IF-ELSE statements evaluate the specified condition, where the evaluation result is a boolean value – true or false. If the condition results in true, then the branch specified under the IF condition gets executed. If the condition results in false, then the ELSE branch is executed. It is planned to integrate ELSEIF branches in the near future.

num x = 4;
if (p1.RelativeStrength > x AND NOT(p1.Sequence == "ATCG")) {
p1.Sequence = "TCGA";
} else {
p1.Sequence = "CGAT";
}
if (on(d1, d2, d3) (r1 AND r2) OR (NOT(r2 OR r3))) {
RBS rbs3("GATC");
} else {
RBS rbs3("ATCG");
}

Loops

To iterate over a set of synthetic biological components or to repeat an execution multiple times. Equivalent to Java, Eugene offers three variations of loops – FOR, WHILE, and DO-WHILE. Loops may be nested too.

FOR Loops

FOR loops take three arguments which are separated by a semicolon – the initialization, condition, and incrementation arguments.
First, the initialization arguments is executed, followed by an evaluation of the condition argument. If the FOR loop condition evaluates to true, the body of the loop gets executed. After the body's execution, the third argument of the FOR loop, i.e. the incrementation argument, is executed, and the loop condition is evaluated again. The FOR loop terminates if the evaluation of the loop conditions evaluates to false.

// print the numbers from 1 to 10
for(num i=1; i<=10; i++) {
print(“i:”, i);
}
// now, let's print the values of the arrayNum2 array
// in reverse order
for(num X=arrayNum2.size()-1; X>=0; X--) {
print(“arrayNum[“,X,”]:”, arrayNum2[X]);
}
WHILE Loops

WHILE loops evaluate a specified conditional expression, and while the evaluation returns true, the specified statements in the loop body will be executed.

// print the numbers from 1 to 10 using a WHILE loop
num i=1;
while(i<=10) {
print(“i: ”,i);
i++;
}
// now, let's print the values of the arrayNum2 array
// in reverse order by using a WHILE loop
num X=arrayNum2.size()-1;
while(X>=0) {
print(“arrayNum[“,X,”]:”, arrayNum2[X]);
X--;
}
DO-WHILE Loops

Different to the WHILE loop, the DO-WHILE loops evaluates the specified condition after the execution of the loop's body instead of before the execution of the loop body. Therefore, the statements within the DO block are always executed at least once.

// print the numbers from 1 to 10 using a DO-WHILE loop
num i=1;
do {
print(“i: ”, i);
i++;
} while(i<=10);
// now, let's print the values of the arrayNum2 array
// in reverse order by using a DO-WHILE loop
num X=arrayNum2.size()-1;
do {
print(“arrayNum[“,X,”]: ”,arrayNum2[X]);
X--;
} while(X>=0);
